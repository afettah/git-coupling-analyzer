# Task: File & Folder Details Panel with Git Integration

**Status:** Proposed  
**Priority:** High  
**Epic:** Repository Exploration & Insights  
**Last Updated:** January 31, 2026

---

## Problem Statement

Users exploring the folder tree can see visual hints but lack the ability to:
- Deep-dive into file/folder statistics without leaving the tree view
- Visualize change history over time with interactive charts
- Open files directly in the remote git repository (GitHub, GitLab, Azure DevOps)
- Navigate to specific commits in the remote repository
- Compare activity across different time periods

This limits the tool's usefulness for code archaeology, ownership analysis, and architectural decision-making.

---

## Goals

1. **Rich Details Panel** â€” Open file/folder details in a dedicated tab with comprehensive statistics
2. **Interactive Charts** â€” Multiple visualization options for change history and patterns
3. **Git Remote Integration** â€” Auto-detect and configure remote URLs for direct linking
4. **Context Menu Actions** â€” Quick actions from tree nodes (open details, open in repo, etc.)
5. **Commit Navigation** â€” Browse and link to commits in the remote repository

---

## Design Decisions

### Remote URL Auto-Detection

**Decision:** Automatically extract remote URL from git config on project creation.

```bash
# Detection command
git -C /path/to/repo remote get-url origin
```

**URL Transformation Rules:**

| Git Remote Format | Transformed Web URL |
|-------------------|---------------------|
| `git@github.com:org/repo.git` | `https://github.com/org/repo` |
| `https://github.com/org/repo.git` | `https://github.com/org/repo` |
| `git@gitlab.com:org/repo.git` | `https://gitlab.com/org/repo` |
| `https://dev.azure.com/org/project/_git/repo` | `https://dev.azure.com/org/project/_git/repo` |
| `org@vs-ssh.visualstudio.com:v3/org/project/repo` | `https://dev.azure.com/org/project/_git/repo` |

**Storage:** Store in `repo_meta` table:
- `git_remote_url` â€” Original remote URL
- `git_web_url` â€” Transformed web URL
- `git_provider` â€” Detected provider (github, gitlab, azure_devops, bitbucket, custom)
- `git_default_branch` â€” Default branch name (main, master, etc.)

### URL Building Patterns

| Provider | File URL | Commit URL | Blame URL |
|----------|----------|------------|-----------|
| **GitHub** | `{base}/blob/{branch}/{path}` | `{base}/commit/{sha}` | `{base}/blame/{branch}/{path}` |
| **GitLab** | `{base}/-/blob/{branch}/{path}` | `{base}/-/commit/{sha}` | `{base}/-/blame/{branch}/{path}` |
| **Azure DevOps** | `{base}?path=/{path}&version=GB{branch}` | `{base}/commit/{sha}` | `{base}?path=/{path}&version=GB{branch}&_a=contents&line=1&lineEnd=1&lineStartColumn=1&lineEndColumn=1&lineStyle=plain` |
| **Bitbucket** | `{base}/src/{branch}/{path}` | `{base}/commits/{sha}` | `{base}/annotate/{branch}/{path}` |

### URL Builder Utility

```typescript
// utils/gitUrls.ts

type GitProvider = 'github' | 'gitlab' | 'azure_devops' | 'bitbucket' | 'custom';

interface UrlTemplates {
  file: string;
  commit: string;
  blame: string;
}

const PROVIDER_TEMPLATES: Record<GitProvider, UrlTemplates> = {
  github: {
    file: '{base}/blob/{branch}/{path}',
    commit: '{base}/commit/{sha}',
    blame: '{base}/blame/{branch}/{path}',
  },
  gitlab: {
    file: '{base}/-/blob/{branch}/{path}',
    commit: '{base}/-/commit/{sha}',
    blame: '{base}/-/blame/{branch}/{path}',
  },
  azure_devops: {
    file: '{base}?path=/{path}&version=GB{branch}',
    commit: '{base}/commit/{sha}',
    blame: '{base}?path=/{path}&version=GB{branch}&_a=contents',
  },
  bitbucket: {
    file: '{base}/src/{branch}/{path}',
    commit: '{base}/commits/{sha}',
    blame: '{base}/annotate/{branch}/{path}',
  },
  custom: {
    file: '{base}/blob/{branch}/{path}',
    commit: '{base}/commit/{sha}',
    blame: '{base}/blame/{branch}/{path}',
  },
};

export function buildFileUrl(config: GitRemoteConfig, path: string): string | null {
  if (!config.web_url) return null;
  const template = config.templates?.file || PROVIDER_TEMPLATES[config.provider].file;
  return template
    .replace('{base}', config.web_url)
    .replace('{branch}', config.default_branch)
    .replace('{path}', encodeURIComponent(path));
}

export function buildCommitUrl(config: GitRemoteConfig, sha: string): string | null {
  if (!config.web_url) return null;
  const template = config.templates?.commit || PROVIDER_TEMPLATES[config.provider].commit;
  return template
    .replace('{base}', config.web_url)
    .replace('{sha}', sha);
}

export function buildBlameUrl(config: GitRemoteConfig, path: string): string | null {
  if (!config.web_url) return null;
  const template = config.templates?.blame || PROVIDER_TEMPLATES[config.provider].blame;
  return template
    .replace('{base}', config.web_url)
    .replace('{branch}', config.default_branch)
    .replace('{path}', encodeURIComponent(path));
}

// Auto-detect provider from URL
export function detectProvider(remoteUrl: string): GitProvider {
  if (remoteUrl.includes('github.com')) return 'github';
  if (remoteUrl.includes('gitlab.com') || remoteUrl.includes('gitlab')) return 'gitlab';
  if (remoteUrl.includes('dev.azure.com') || remoteUrl.includes('visualstudio.com')) return 'azure_devops';
  if (remoteUrl.includes('bitbucket.org')) return 'bitbucket';
  return 'custom';
}

// Transform git remote URL to web URL
export function transformToWebUrl(remoteUrl: string): string {
  // git@github.com:org/repo.git â†’ https://github.com/org/repo
  // https://github.com/org/repo.git â†’ https://github.com/org/repo
  
  let url = remoteUrl
    .replace(/^git@([^:]+):/, 'https://$1/')
    .replace(/\.git$/, '');
  
  // Azure DevOps SSH format
  if (url.includes('vs-ssh.visualstudio.com')) {
    // org@vs-ssh.visualstudio.com:v3/org/project/repo
    const match = url.match(/([^@]+)@vs-ssh\.visualstudio\.com:v3\/([^/]+)\/([^/]+)\/(.+)/);
    if (match) {
      return `https://dev.azure.com/${match[2]}/${match[3]}/_git/${match[4]}`;
    }
  }
  
  return url;
}
```
